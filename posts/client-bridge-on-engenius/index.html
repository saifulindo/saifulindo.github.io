Client Bridge on EnGenius

http://saifulindo.github.io/posts/client-bridge-on-engenius/

Published: Sept 20, 2014

Published: Sept 20, 2014

Sebenarnya Client bridge ini bukan pokok pembahasan akan tetapi bagian dari sebuah Materi Pelajaran “Men-Diagnosis Permasalahan Perangkat yang Tersambung Jaringan Berbasis Luas” dengan Kode Modul NTW.MNT.201.(2).A yang penulis ajarkan di kelas Teknik Komputer dan Jaringan, yang dibagi menjadi tiga kompetensi dasar (1) Mengidentifikasi masalah fungsionalitas jaringan pada perangkat melalui gejala yang muncul, (2) Memilih masalah berdasarkan kelompoknya, (3) Menganalisa DHCP Client. Jadi topologi yang dirumuskan harus dapat mewakili kompetensi dasar tersebut.

Client Bridge digunakan untuk menghubungkan dua jaringan kabel menggunkan dua router nirekabel tanpa membuat jembatan, sedangkan EnGenius merupakan perangkat access point yang digunakan untuk melakaukan client bridge. Untuk melakukan client bridge tidak harus menggunkan EnGenius, bisa menggunkan merek lain seperti Cisco, Tp-link dan lainnya yang mendukung client bridge.

Kegunaan Client bridge akan menjadi jelas dengan melihat topologi berikut :

Beberapa tahapan untuk melakukan client bridge :

  • Merumuskan topologi jaringan
  • Konfigurasi EnGenius NH200
  • Konfigurasi Router Server
  • Konfigurasi WirelessLAN D-Link
  • Uji Koneksi
  • Konfigurasi Wifi Client

Yang perlu diperhatikan dalam melakukan beberapa tahapan tersebut adalah (1) ketika merumuskan topologi jaringan dalam melakukan konfigurasi IP Address menyesuaikan penggunaan IP Address di lokasi praktik, (2) Pada router server apabila menambahkan LAN Card yang berganti-ganti gunakan perintah :

# ifconfig ethn up 
n diisi 1 sampai tidak ditentukan

Pastikan tidak ada pesan kesalahan, (3) WirelessLAN bisa digantikan menjadi perangkat lain seperti Switch, (4) Pastikan Client dapat melakukan ping ke google dan AP Kantor :

# ping google.com 
# ping 192.168.100.1

IP Address Client dapat di setting secara automaticaly apabila WirelessLAN DHCP-nya di enable, atau router server mengaktifkan DHCP Server kemudian WirelessLAN di konfigurasi sebagai bridge.
